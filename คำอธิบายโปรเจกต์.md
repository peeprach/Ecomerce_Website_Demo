# อธิบายโปรเจกต์ CSS Mall ทั้งหมด

---

## ภาพรวมโปรเจกต์

CSS Mall คือเว็บแอปพลิเคชัน **ตลาดซื้อขายออนไลน์ (E-Commerce Marketplace)**
ที่ทำด้วยเทคโนโลยีหลัก 4 ตัว:

| เทคโนโลยี | หน้าที่ |
|---|---|
| **HTML** | กำหนดโครงสร้างหน้าเว็บ (ปุ่ม, ฟอร์ม, ตาราง) |
| **CSS** | ตกแต่งหน้าตา (สี, ขนาด, ตำแหน่ง) |
| **JavaScript** | ทำให้เว็บทำงานได้ (ดึงข้อมูล, คลิกปุ่ม) |
| **Firebase** | เซิร์ฟเวอร์ฐานข้อมูล + ระบบล็อกอิน (ไม่ต้องเขียน backend เอง) |

## Firebase คืออะไร? ทำงานยังไง?

Firebase คือบริการของ Google ที่รวม **ฐานข้อมูล + ระบบล็อกอิน + พื้นที่เก็บไฟล์**
ไว้ในที่เดียว เว็บไซต์นี้ใช้ Firebase แทนการเขียน backend server เองทั้งหมด

ทุกหน้าที่ใช้ Firebase จะต้องโหลด SDK ผ่าน `<script>` tag ใน `<head>` ก่อน:

```html
<script src="https://www.gstatic.com/firebasejs/9.17.1/firebase-app-compat.js"></script>
<!-- โหลดตัว Firebase หลัก (core) มาจาก server ของ Google -->

<script src="https://www.gstatic.com/firebasejs/9.17.1/firebase-auth-compat.js"></script>
<!-- โหลดโมดูลจัดการล็อกอิน/สมัครสมาชิก -->

<script src="https://www.gstatic.com/firebasejs/9.17.1/firebase-firestore-compat.js"></script>
<!-- โหลดโมดูลฐานข้อมูล Firestore -->
```

จากนั้นทุกหน้าจะมีโค้ดชุดนี้เพื่อ "เชื่อมต่อ" กับโปรเจกต์ Firebase บน cloud:

## 2. หน้า Login (`login.html`) — ระบบเข้าสู่ระบบ

หน้านี้เชื่อมกับ **Firebase Authentication** ซึ่งจัดการล็อกอิน/สมัครสมาชิกให้ทั้งหมด

### HTML ฟอร์ม

```html
<form id="login-form">
<!-- สร้างฟอร์ม id="login-form" เพื่อให้ JavaScript หาเจอด้วย getElementById -->

  <input type="email" id="email" placeholder="Enter your email" required>
  <!-- ช่องกรอก email
       type="email"  = browser ตรวจว่าต้องมี @ ด้วย
       placeholder   = ข้อความสีเทาที่แสดงตอนยังไม่พิมพ์
       required      = ส่งฟอร์มไม่ได้ถ้าว่าง -->

  <input type="password" id="password" placeholder="Enter your password" required>
  <!-- type="password" = ซ่อนตัวอักษรเป็นจุดกลม -->

  <button type="submit" class="btn-login">LOG IN</button>
  <!-- type="submit" = กดแล้ว trigger event 'submit' ของฟอร์ม -->
</form>
```

### JavaScript — ล็อกอินด้วย Email/Password

```js
document.getElementById('login-form').addEventListener('submit', (e) => {
// หาฟอร์มที่มี id="login-form" แล้วรอ event 'submit' (กดปุ่ม LOG IN)
// (e) คือ event object ที่ browser ส่งมาให้

  e.preventDefault();
  // หยุดการทำงานปกติของฟอร์ม (ปกติจะ reload หน้า)
  // เราจัดการเองแทนด้วย Firebase

  const email = document.getElementById('email').value.trim();
  // อ่านค่าจากช่องกรอก email
  // .value  = ข้อความที่ผู้ใช้พิมพ์
  // .trim() = ตัดช่องว่างหัว-ท้ายออก เช่น "  user@mail.com  " → "user@mail.com"

  const password = document.getElementById('password').value;
  // อ่านค่า password (ไม่ต้อง .trim() เพราะ password อาจมีช่องว่างตั้งใจ)

  auth.signInWithEmailAndPassword(email, password)
  // ส่ง email+password ไปให้ Firebase ตรวจสอบ
  // Firebase จะเช็คกับฐานข้อมูลผู้ใช้บน cloud
  // คืนค่าเป็น Promise (ทำงานแบบ async = ไม่รอผล แต่รับผ่าน .then/.catch)

    .then((userCredential) => {
    // ถ้าล็อกอินสำเร็จ Firebase จะเรียก .then()
    // userCredential คือข้อมูลผู้ใช้ที่ล็อกอินสำเร็จ

      loginMessage.style.color = "green";
      loginMessage.innerText = "Login successful! Redirecting...";

      setTimeout(() => {
        window.location.href = "shop.html";
      }, 1500);
      // setTimeout = รอ 1500ms (1.5 วินาที) แล้วพาไปหน้า shop.html
      // window.location.href = วิธีเปลี่ยนหน้าใน JavaScript
    })
    .catch((error) => {
    // ถ้าล็อกอินไม่สำเร็จ (password ผิด, user ไม่มี) Firebase เรียก .catch()
    // error.message คือข้อความอธิบายปัญหา เช่น "Wrong password"

      loginMessage.style.color = "red";
      loginMessage.innerText = error.message;
    });
});
```

### JavaScript — ล็อกอินด้วย Google

```js
document.querySelector('.btn-google').addEventListener('click', () => {
// querySelector หาปุ่มที่มี class="btn-google"
// addEventListener('click') = รอผู้ใช้คลิก

  const provider = new firebase.auth.GoogleAuthProvider();
  // สร้าง provider = ตัวเชื่อมกับระบบล็อกอิน Google
  // Firebase รองรับหลาย provider: Google, Apple, Facebook, Phone ฯลฯ

  auth.signInWithPopup(provider)
  // เปิด popup หน้าต่าง Google เพื่อให้ผู้ใช้เลือก account
  // Firebase จัดการทุกอย่างให้ เราแค่เรียก function นี้

    .then(() => {
      window.location.href = "shop.html";
      // ล็อกอินสำเร็จ → พาไปหน้า shop
    })
    .catch((error) => {
      document.getElementById('login-message').innerText = error.message;
    });
});
```

### JavaScript — ล็อกอินด้วยเบอร์โทร

```js
window.recaptchaVerifier = new firebase.auth.RecaptchaVerifier(
  'recaptcha-container',
  { size: 'invisible' }
);
// สร้าง reCAPTCHA แบบ invisible (ผู้ใช้ไม่เห็น แต่ Firebase ต้องการ)

auth.signInWithPhoneNumber(phoneNumber, window.recaptchaVerifier)
// ส่ง OTP ไปยังเบอร์โทร

  .then((confirmationResult) => {

    return confirmationResult.confirm(otp);
    // ยืนยัน OTP กับ Firebase
  })
  .then(() => {
    window.location.href = "shop.html";
  });
```

## 3. หน้า Home (`home.html`) — หน้าหลัก

หน้านี้มีหน้าที่หลัก 2 อย่าง: **ตรวจว่า login อยู่ไหม** และ **แสดง banner**

```js
auth.onAuthStateChanged(user => {
// Firebase จะเรียก function นี้ทุกครั้งที่ "สถานะ login เปลี่ยน"
// เช่น เปิดหน้าใหม่, login, logout
// user = ข้อมูลผู้ใช้ถ้า login อยู่ / null ถ้ายังไม่ login

  if (!user) {
    window.location.href = "login.html";
    // ถ้าไม่ได้ login อยู่ → พาไปหน้า login ทันที
    // นี่คือ "การป้องกันหน้า" (route guard) ที่ทุกหน้าหลัง login ต้องมี
  }
});

document.getElementById('logout-btn').addEventListener('click', async (e) => {

  await auth.signOut();
  // auth.signOut() = ล็อกเอาต์จาก Firebase
  // await = รอให้ล็อกเอาต์เสร็จก่อน

  window.location.href = "login.html";
```

### JavaScript — ดึงสินค้าจาก Firestore แบบ Real-time

```js
let allProducts = [];
// ตัวแปรเก็บรายการสินค้าทั้งหมด เริ่มต้นเป็น array ว่าง

db.collection("products").orderBy("name").onSnapshot(snapshot => {
// db.collection("products")  = เข้าถึง collection ชื่อ "products" ใน Firestore
//                               คิดเหมือนตาราง/โฟลเดอร์ที่เก็บสินค้าทุกชิ้น
// .orderBy("name")            = เรียงตามชื่อ A-Z
// .onSnapshot()               = ฟัง real-time!
//                               ถ้ามีคนเพิ่ม/ลบ/แก้สินค้า หน้าเว็บจะอัปเดตทันที
//                               ต่างจาก .get() ที่ดึงแค่ครั้งเดียว

  allProducts = snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }));
  // snapshot.docs  = array ของทุก document ใน collection
  // .map()         = วนซ้ำแปลงทุก doc เป็น object ใหม่
  // doc.id         = รหัสเฉพาะของสินค้า (Firestore สร้างให้อัตโนมัติ)
  // doc.data()     = ข้อมูลสินค้า เช่น { name: "Nike", price: 500 }
  // ...doc.data()  = spread operator = คัดลอกทุก field จาก data() มาใส่ด้วย
  // ผลลัพธ์: { id: "abc123", name: "Nike", price: 500, imageUrl: "..." }

  renderProducts(allProducts);
  // เรียก function สร้างการ์ดสินค้าขึ้นหน้าจอ
});
```

### JavaScript — สร้างการ์ดสินค้า

```js
function renderProducts(products) {
  productGrid.innerHTML = "";
  // ล้าง grid เก่าออกก่อน แล้วค่อยใส่ใหม่
  // ถ้าไม่ล้าง สินค้าจะซ้อนกัน

  products.forEach(p => {
  // วนซ้ำทุกสินค้าใน array

    const card = document.createElement("div");
    // สร้าง element <div> ใหม่ใน memory (ยังไม่แสดงบนหน้า)

    card.classList.add("product-card");
    // ใส่ class "product-card" เพื่อให้ CSS จัดสไตล์ได้

    card.innerHTML = `
      <img src="${p.imageUrl || 'images/placeholder.jpg'}" alt="${p.name}" />
      <!-- รูปสินค้า
           p.imageUrl || 'images/placeholder.jpg' = ถ้าไม่มีรูป ใช้รูป placeholder แทน
           ${...} = template literal = แทรกค่าตัวแปรลงใน HTML string -->

      <h3>${p.name}</h3>
      <p class="price">$${p.price}</p>

      <button onclick="addToCart('${p.id}')">Add to Cart</button>
      <!-- ปุ่ม Add to Cart ส่ง id สินค้าเข้า function -->

      <button onclick="buyNow('${p.id}')">Buy Now</button>
      <!-- ปุ่ม Buy Now = add to cart แล้วไป cart.html เลย -->
    `;

    productGrid.appendChild(card);
    // เพิ่ม card ที่สร้างแล้วเข้าไปใน product-grid บนหน้าเว็บ
  });

  resultCount.innerText = `${products.length} products found`;
  // แสดงจำนวนสินค้า เช่น "12 products found"
}
```

### JavaScript — เพิ่มสินค้าลงตะกร้า

```js
function addToCart(productId) {
  if (!currentUser) { showToast("Please log in first."); return; }
  // ตรวจก่อน ถ้าไม่ได้ login แสดง toast แล้วหยุดทำงาน
  // return = ออกจาก function ทันที

  const ref = db.collection("carts").doc(currentUser.uid)
                .collection("items").doc(productId);
  // สร้าง reference ไปยัง path ใน Firestore:
  //   carts → [uid ของผู้ใช้] → items → [id สินค้า]
  // currentUser.uid = รหัสเฉพาะของผู้ใช้ที่ login อยู่ (Firebase สร้างให้อัตโนมัติ)
  // ตัวอย่าง path: carts/abc123xyz/items/nike001

  ref.get().then(doc => {
  // .get() = ดึงข้อมูลจาก Firestore ครั้งเดียว (ไม่ real-time)

    if (doc.exists) {
      return ref.update({ quantity: doc.data().quantity + 1 });
      // ถ้าสินค้านี้มีในตะกร้าแล้ว → เพิ่ม quantity +1
      // .update() = อัปเดตแค่ field ที่ระบุ ไม่ลบ field อื่น
    } else {
      return ref.set({ quantity: 1 });
      // ถ้ายังไม่มี → สร้าง document ใหม่ quantity = 1
      // .set() = สร้างหรือเขียนทับทั้งหมด
    }
  })
  .then(() => showToast("Added to cart!"))
  // ถ้าสำเร็จ แสดง toast (แทน alert popup เดิม)

  .catch(err => showToast(err.message));
  // ถ้า error แสดง error message
}
```

```js
function renderCartItems() {

  // ดึงทุก item ในตะกร้า

    cartSnapshot.forEach(docSnap => {
    // วนทุก item ในตะกร้า

      const item = docSnap.data();
      // ดึงข้อมูล item เช่น { quantity: 2, productId: "nike001" }

        const product = productSnap.data();

      });

    Promise.all(promises).then(() => {
    // Promise.all รอให้ทุก promise เสร็จก่อน
    // เพราะดึงสินค้าหลายชิ้นพร้อมกัน ต้องรอทุกชิ้นเสร็จ แล้วถึงคำนวณ total
```

### JavaScript — อัปเดต Quantity

```js
function updateQuantity(itemDocId, change) {
// change = +1 หรือ -1

  itemRef.get().then(doc => {
    let newQty = doc.data().quantity + change;
    // คำนวณ quantity ใหม่

    if (newQty < 1) newQty = 1;
    // ป้องกัน quantity ติดลบหรือเป็น 0

    itemRef.update({ quantity: newQty }).then(() => {
    // บันทึก quantity ใหม่ลง Firestore

      document.getElementById(`qty-${itemDocId}`).textContent = newQty;
      // อัปเดตตัวเลข quantity บนหน้าจอ โดยไม่ต้อง reload ทั้งหน้า
      // ใช้ template literal `qty-${itemDocId}` เพื่อหา element ที่ถูกต้อง

      document.getElementById(`subtotal-${itemDocId}`).textContent =
        `$${product.price * newQty}`;
      // อัปเดตราคา subtotal ของสินค้าชิ้นนี้
    });
  });
}
```

## 8. `firestore.rules` — กฎความปลอดภัย

ไฟล์นี้กำหนดว่า **ใครอ่าน/เขียนอะไรได้บ้าง** ใน Firestore
ถ้าไม่มีไฟล์นี้หรือกฎหมดอายุ ฐานข้อมูลจะถูกปิดทั้งหมด

```js
rules_version = '2';
// กำหนดเวอร์ชันของ Firestore rules syntax

service cloud.firestore {
// บอกว่ากฎนี้ใช้กับ Firestore (มีบริการอื่นด้วย เช่น Storage)

  match /databases/{database}/documents {
  // match = กำหนด pattern ของ path ที่จะใช้กฎนี้
  // {database} = wildcard รับชื่อ database ใดก็ได้

    match /products/{productId} {
    // กฎสำหรับ collection "products"
    // {productId} คือ wildcard = รับรหัส document ใดก็ได้

      allow read: if true;
      // อนุญาตให้ทุกคนอ่านสินค้าได้ (แม้ไม่ login)

      allow create: if request.auth != null
                    && request.resource.data.sellerId == request.auth.uid;
      // สร้างสินค้าได้ต้องเงื่อนไข 2 ข้อ:
      // 1. request.auth != null = ต้อง login ก่อน
      // 2. sellerId ในข้อมูลที่จะบันทึก ต้องตรงกับ uid ของคนที่ login
      //    (ป้องกันคนอื่นสร้างสินค้าในนามผู้ขายคนอื่น)

      allow update, delete: if request.auth != null
                            && resource.data.sellerId == request.auth.uid;
      // แก้ไข/ลบได้ต้องเป็นเจ้าของสินค้านั้น
      // resource.data = ข้อมูลที่อยู่ใน Firestore อยู่แล้ว (ก่อนแก้)
    }

    match /carts/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
      // อ่าน/เขียนตะกร้าได้เฉพาะเจ้าของตะกร้า
      // userId คือ wildcard ที่จับจาก path "carts/userId"
      // request.auth.uid == userId = UID ที่ login ต้องตรงกับ path

      match /items/{itemId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
        // sub-collection items ใช้กฎเดียวกัน
      }
    }

    match /orders/{orderId} {
      allow create: if request.auth != null
                    && request.resource.data.userId == request.auth.uid;
      // สร้าง order ได้เฉพาะ login แล้ว และ userId ต้องตรงกับ uid ตัวเอง

      allow read: if request.auth != null
                  && resource.data.userId == request.auth.uid;
      // อ่าน order ได้เฉพาะเจ้าของ order นั้น
    }
  }
}
```
